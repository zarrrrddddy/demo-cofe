# Demo Loyalty Bot v2 — з "вау-ефектом" (Telegram Mini App)

Той самий бот, що і раніше, плюс головна фішка: справжня візуальна
картка лояльності, яка відкривається прямо всередині Telegram —
анімований прогрес-бар, список нагород з іконками, історія операцій,
конфеті при розблокуванні нагороди. Виглядає як окремий застосунок,
а не просто бот з текстом.

## Що нового порівняно з v1

- `/card` (і кнопка "🎴 Відкрити картку лояльності") відкриває
  Telegram Mini App — стильний темний інтерфейс з анімаціями
- Дані підтягуються через захищений API-ендпоінт (`/api/me`),
  захищений офіційною перевіркою підпису Telegram (`initData`) —
  ніхто сторонній не побачить чужий баланс
- Все залишається в одному процесі: бот і веб-сервер міні-аппу
  працюють разом (через `aiohttp`, який вже йде разом з `aiogram`)

## ⚠️ Головна умова: потрібен публічний HTTPS-домен

Telegram Mini App **не відкриється на localhost** — тільки на
справжньому https-домені. Тому:

1. Задеплой бота на Railway як завжди
2. У Railway: **Settings → Networking → Generate Domain** (якщо ще не
   згенеровано) — отримаєш домен виду
   `https://demo-loyalty-bot-production.up.railway.app`
3. Railway автоматично прокидає цей домен у змінну
   `RAILWAY_PUBLIC_DOMAIN` — бот підхопить її сам, нічого додатково
   вказувати не треба
4. Якщо з якоїсь причини не підхопилось — постав вручну змінну
   `PUBLIC_URL=https://твій-домен.up.railway.app`

Поки `PUBLIC_URL`/`RAILWAY_PUBLIC_DOMAIN` не задані — кнопка міні-аппу
просто не показується, бот тихо працює по-старому (текстовий баланс),
нічого не ламається.

## Важливо: Railway має слухати правильний порт

Railway сам підставляє змінну `PORT` — бот вже налаштований її
підхоплювати (`os.environ.get("PORT", "8080")`), нічого міняти не
треба. Головне — щоб у Railway у **Settings → Networking** був
включений публічний домен саме для цього сервісу.

## Встановлення (локально, без міні-аппу)

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

export BOT_TOKEN="токен_від_BotFather"
export ADMIN_IDS="твій_telegram_id"

python bot.py
```

Локально бот працюватиме повністю, окрім кнопки картки лояльності —
вона з'явиться тільки після деплою з HTTPS-доменом.

## Структура файлів

```
bot.py              — бот + вбудований веб-сервер міні-аппу
static/index.html   — сама сторінка картки лояльності (HTML/CSS/JS)
requirements.txt
```

## Що можна кастомізувати пізніше

- Кольори/шрифти — у `static/index.html`, змінні `--accent`, `--accent2` у CSS
- `REWARDS` у `bot.py` — реальні назви й вартість напоїв
- `CAFE_NAME` — назва закладу (змінна середовища)
