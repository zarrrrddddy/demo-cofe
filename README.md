# Demo Loyalty Bot — універсальний бот-приклад для показу кав'ярням

Той самий функціонал, що і в проєкті під конкретне кафе, але без
прив'язки до бренду — можна показувати будь-якому закладу як демо,
не пояснюючи "це чужий проєкт".

## Швидкий запуск

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

export BOT_TOKEN="токен_нового_бота_від_BotFather"
export ADMIN_IDS="твій_telegram_id"
export CAFE_NAME="Demo Coffee"   # необов'язково, за замовчуванням "Demo Coffee"

python bot.py
```

## Важливо: створи ОКРЕМОГО бота в @BotFather

Не використовуй той самий токен, що і для Leenalchi — заведи нового
бота спеціально для демонстрацій (наприклад `@your_loyalty_demo_bot`),
щоб:
- можна було показувати всім однаковий приклад
- не змішувати тестові дані з реальними клієнтами Leenalchi
- пізніше легко перейменувати/перебрендувати під конкретного клієнта

## Деплой

Той самий процес, що і раніше — Railway, Start Command `python bot.py`,
Volume для збереження бази між деплоями (опційно, для демо не критично).

## Команди

**Клієнт**
- `/start` — реєстрація (кнопка "Поділитися номером")
- `/balance` — баланс і прогрес до винагороди
- `/history` — останні операції

**Адмін**
- `/admin` — довідка
- `/find +380XXXXXXXXX` або останні 4 цифри — знайти клієнта
- `/add` / `/sub` — нарахувати/списати бали
- `/register` — зареєструвати клієнта вручну
- `/broadcast` — розсилка фото-новини всім клієнтам

## Що поміняти під конкретного клієнта пізніше

- `CAFE_NAME` — назва закладу (змінна середовища)
- `REWARDS` у коді — реальні назви й вартість напоїв
- За бажанням — емодзі й кольори у текстах повідомлень
